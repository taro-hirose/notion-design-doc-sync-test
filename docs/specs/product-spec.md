# Product Spec Doc: ユーザー通知機能

> このファイルはNotionから自動同期されます。直接編集しないでください。

## 経緯

- ユーザーから「重要な更新を見逃してしまう」というフィードバックが多数
- カスタマーサポートへの問い合わせの30%が「通知がない」ことに起因
- 競合サービスでは標準機能として提供されている

## AsIs（現状）

### 現状のフロー

1. ユーザーがアプリを開く
2. 手動でダッシュボードを確認
3. 新着情報があれば確認する

### 課題・ペインポイント

- ユーザーが能動的にアプリを開かないと情報を得られない
- 重要な締め切りや更新を見逃すことが多い
- エンゲージメント率が低下している

## ToBe（目指す姿）

### 理想のフロー

1. 重要なイベントが発生
2. システムが自動で通知を送信
3. ユーザーがプッシュ通知/メールで受け取る
4. 通知からワンタップでアプリの該当画面へ遷移

### 期待される成果

- DAU 20%向上
- カスタマーサポート問い合わせ 30%削減
- ユーザー満足度スコア向上

## 検討

### 案A: プッシュ通知のみ

**メリット**
- 実装がシンプル
- リアルタイム性が高い

**デメリット**
- 通知をOFFにしているユーザーに届かない
- 過度な通知はアンインストールにつながる

### 案B: プッシュ通知 + メール + アプリ内通知

**メリット**
- 複数チャネルでリーチ可能
- ユーザーが好みの方法を選択できる

**デメリット**
- 実装コストが高い
- 通知設定のUIが複雑になる

### 決定

案Bを採用。理由: ユーザーの多様なニーズに対応し、長期的なエンゲージメント向上を優先
